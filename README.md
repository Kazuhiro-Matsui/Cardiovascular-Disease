# 心血管疾患リスク予測モデル：GAM/BAM による解釈可能ヘルスケア AI

> **TL;DR** — 7 万件の健康診断データから心血管疾患の陽性確率を予測する
> 解釈可能モデルを R + `mgcv::bam`（一般化加法モデルの大規模データ向け拡張）で実装。
> 5-fold CV で **Accuracy 72.97% / Recall 75.45% / F1 73.40%**、
> テスト評価で **Accuracy 72.60% / Recall 75.02%**。
> 95% 信頼区間によるフェイルセーフ判定モードを実装し、産業医・健診業務の
> **一次スクリーニング AI** としての運用フローを設計した。

## 1. 背景：なぜこの課題を選んだか

大企業の産業医・産業保健師は数千〜数万人の健診データを一人ずつ目視確認しており、
再検査・要検査対象者の抽出に膨大な工数を割いている（知人の産業保健師へのヒアリングより）。

健診データから陽性リスクの高い従業員を AI で**瞬時に**スクリーニングできれば、
産業医の判断業務を**本質的な高リスク症例**に集中させることができ、
医療現場や企業におけるヘルスケア業務の効率化に直結する。

なお、診断データさえあれば原理的にはどんな疾患リスクも予測可能だが、
学習データに**目的変数（正解ラベル）が存在する必要がある**。
ラベルがない場合はクラスタリング等の潜在変数モデルが必要となり、別アプローチになる。

> 💡 **本プロジェクトの問い**：
> 「説明可能性を保ったまま、このデータセットの理論上限精度（73–74%）に
> R 言語の伝統的統計モデルで到達できるか？」

## 2. データ

- **Source**: [Kaggle - Cardiovascular Disease Dataset](https://www.kaggle.com/datasets/sulianova/cardiovascular-disease-dataset/data)
- **Samples**: 70,000 → クレンジング後 **68,610 件**
- **Target**: `cardio` (0 = 陰性, 1 = 陽性)
- **取得手順**: [`data/README.md`](data/README.md) 参照

### 前処理サマリ

| 操作         | 内容                                                                          |
| ------------ | ----------------------------------------------------------------------------- |
| 範囲フィルタ | `ap_hi ∈ [70, 250]`, `ap_lo ∈ [40, 150]`, `ap_hi > ap_lo`, 身長 [120, 210] cm, 体重 [30, 200] kg |
| 派生特徴量   | `age_years = age / 365.25`, `bmi`, `pulse_pressure = ap_hi - ap_lo`           |
| factor 化    | `gender`, `cholesterol`, `gluc`, `smoke`, `alco`, `active`, `cardio`           |

実装: [`src/preprocess.R`](src/preprocess.R)

## 3. 手法：なぜ BAM (mgcv) を選んだか

### 3.1 モデル選定ロジック

| 選択肢                       | 採用 | 理由                                                       |
| ---------------------------- | :--: | ---------------------------------------------------------- |
| LightGBM / XGBoost           |  ❌  | 高精度だが医療領域の説明責任に弱い                         |
| Logistic Regression          |  ❌  | 年齢・血圧の非線形効果を表現できない                       |
| **BAM (Big GAM)**            |  ✅  | スプライン非線形 + p 値 / 信頼区間で**統計的に説明可能**   |
| Bayesian Hierarchical (Stan) |  △  | 7 万件規模では計算コスト面で本番運用に不向き、将来検討     |

### 3.2 モデル定式化

```r
cardio ~ gender +
  s(age_years,      by = gender) +   # 年齢効果を性別ごとに非線形推定
  s(bmi,            by = gender) +
  s(pulse_pressure, by = gender) +
  s(ap_hi,          by = gender) +
  cholesterol + gluc + smoke + alco + active
```

`by = gender` で**性別ごとに別スプラインを推定**することにより、
「男女で動脈硬化リスクの年齢進行が異なる」という医学的知見をモデル構造で表現した。

実装: [`src/train_bam.R`](src/train_bam.R)

### 3.3 検証戦略

- **5-fold Cross Validation**（fold ID を乱数で生成）
- 訓練 80% / テスト 20% ホールドアウト
- **不確実性算出**: `predict(..., type="link", se.fit=TRUE)` でロジット空間の SE を取得し、
  `plogis(fit ± 1.96 × se)` で確率の 95% CI に逆変換

## 5. 考察：単純な数値以上に重要な "バイアスの読み"

### 5.1 性能評価：理論上限への到達

このデータセットは健康診断の基本項目（血液検査の一部・血圧・体型など）しか含まれていないため、
Kaggle コミュニティでは Accuracy の上限は **概ね 73〜74% 程度が限界
（どんな AI を使ってもこれ以上は上がらない）** と言われている。

医療やヘルスケアの分野では、AI の評価指標として **適合率（Precision）よりも
再現率（Recall）が重視されるべき**である（偽陰性を極力なくすことが最優先のため）。

つまり、現在の **Accuracy 72.6% / Recall 75.0%** という数字は、
**このデータセットにおける最高到達点に近い性能**である。

`summary(final_gam_model)` から、**収縮期血圧 (`ap_hi`) のカイ二乗値 (男性 2146, 女性 1373)
が圧倒的に大きく、予測に最も大きな影響を与えている**ことも確認できた。

### 5.2 直感に反する係数の発見

個人的に興味深かったのは、**喫煙・飲酒・高血糖値でリスクが下がる**という
直感に反する現象が起きていることである。

| 変数                     | 係数       | 直感             |
| ------------------------ | ---------: | ---------------- |
| **`smoke1`** (喫煙)      | **−0.1831** ⚠️ | 喫煙はリスク要因のはず |
| **`alco1`** (飲酒)       | **−0.2011** ⚠️ | 飲酒はリスク要因のはず |
| **`gluc3`** (重度高血糖) | **−0.3283** ⚠️ | 重度高血糖は最大リスク要因のはず |

これらの係数がマイナスになっている現象は、**AI が間違えたのではなく**、
今回のデータに以下のような強いバイアス（偏り）が潜んでいることが原因だと考えられる。

### 5.3 理由 A：逆因果関係（治療・行動変容バイアス）

これが最も大きな原因だろう。このデータは「**ある時点での健康診断結果**」の
スナップショットである。

心血管疾患のリスクが高い人（あるいはすでに自覚症状がある人）は、
医師から強く指導され、**「すでにタバコや酒をやめている（`smoke=0, alco=0`）」**
可能性が非常に高い。逆に、健康に自信がある人ほどタバコやお酒を楽しんでいる。

また、血糖値が極端に高い人（`gluc3`）は、すでに**厳しい食事制限や
強力な薬物治療を受けてコントロールしている状態**かもしれない。

モデルはここで、

> 「タバコを吸っている＝比較的健康な証拠」
> 「タバコを吸っていない＝病気だからやめさせられた人」

という**相関関係を拾ってしまった**と考えられる。

### 5.4 理由 B：強い交絡変数（性別）によるマスキング

このデータセットでは、**「喫煙者と飲酒者の圧倒的多数が男性である」**
という強い偏りがある。

モデルの出力を見ると、

- **男性（`gender2`）であるだけで +0.21 のリスクが加算**
- さらに男性に多い高血圧（`s(ap_hi):gender2`, χ² = 1372）でも**大きなリスクが加算**

すでに「男性」と「血圧」で十分に高い疾患リスクが計算されているため、
統計モデルが**数式の帳尻を合わせる際**、残った「喫煙」の要素が
**数学的に押し出されてマイナスに振れてしまった**と考えられる
（多重共線性や **シンプソンのパラドックス** に近い現象）。

### 5.5 結論：このモデルで言えること・言えないこと

| 言えること                                       | 言えないこと                                                   |
| ------------------------------------------------ | -------------------------------------------------------------- |
| 「単純な予測精度は 73%」というスクリーニング性能 | 「**喫煙が無意味**」「**禁酒は意味がない**」といった因果的主張 |
| 健診データから "陽性リスクが高そうな人" を抽出   | このデータから予防医学の介入効果を導出する                     |

まとめると、**単純な予測精度は 73% と高いが、喫煙や飲酒の係数がマイナスになることから、
このデータには行動変容バイアス（病気の人は生活習慣を改善している）が含まれており、
予防医学の因果関係を示す根拠としては使えない**。

**因果と相関を厳密に区別すること**は、医療 AI 開発者に求められる責任である。

## 6. 実運用への提案：二段階フィルター設計

このデモのモデルでは **偽陰性を極力減らすために 0.4 以上を陽性と評価し、再現率を最大化**した。
適合率（Precision）は 0.71 で、つまり**実際には偽陽性が約 30% 含まれる**。

したがって、医療スクリーニングにおける役割分担として、本モデルのような AI は
**病院での最終診断を下すためのものではなく、健康診断のデータから陽性かもしれない人を
拾い上げる（スクリーニングする）**という**一次フィルター**の役割を果たすべきである。

```
[健診データ (全従業員 / 全受診者)]
   │
   ▼
[① AI 一次スクリーニング]   ← 本プロジェクト
   │   ・しきい値 0.4 で Recall 最大化（75-78%）
   │   ・偽陽性 ≈ 30% を許容
   │   ・目的: 産業医・医師の確認対象を絞り込む
   ▼
[② 医師による精密検査 (二次フィルター)]
   │   ・AI がすくい上げた人たちを詳しく調べる
   │   ・本当に病気の人だけを確定診断する
   │   ・ここで初めて Precision を高める作業を行う
   ▼
[確定診断・治療介入]
```

> 💡 **重要な設計思想**：
> AI の役割は「**全員を見る**」を「**高リスク群を見る**」に絞り込むこと。
> Precision を高める作業は AI ではなく **二次フィルター（医師）が担う**。
> この役割分担を明確にすることが、医療 AI を**安全に社会実装する前提条件**である。



## 7. 再現方法

### 環境

- R ≥ 4.2
- パッケージ: `dplyr`, `mgcv`（[`renv.lock`](renv.lock) 参照）
- もしくは Google Colab + `rpy2`（[`notebooks/cardio_bam.ipynb`](notebooks/cardio_bam.ipynb)）

### セットアップ

```bash
git clone https://github.com/Kazuhiro-Matsui/Cardiovascular-Disease.git
cd Cardiovascular-Disease

# データ取得（data/README.md 参照）
kaggle datasets download -d sulianova/cardiovascular-disease-dataset -p ./data/
unzip ./data/cardiovascular-disease-dataset.zip -d ./data/

# R パッケージインストール
R -e 'install.packages(c("dplyr", "mgcv"))'
```

### 実行

```bash
# 1) 前処理の動作確認
Rscript src/preprocess.R

# 2) 5-fold CV
Rscript src/train_bam.R

# 3) 最終評価 + プロット保存
Rscript src/evaluate.R

# 4) 単体テスト
Rscript tests/test_preprocess.R
```

### Jupyter / Colab で実行する場合

```bash
pip install -r requirements.txt
jupyter notebook notebooks/cardio_bam.ipynb
```
